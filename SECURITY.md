# Security Policy

## Supported Versions
Security fixes are applied to the latest state of the `main` branch and the most recent public release.

## Reporting a Vulnerability
Please do not open a public issue for security-sensitive reports.

Preferred path:
- Use GitHub's private vulnerability reporting for this repository, if it is enabled.

Fallback path:
- Contact the maintainer through GitHub before publishing details publicly.

Please include:
- A clear description of the issue
- Steps to reproduce
- Expected impact
- Any proof-of-concept material that helps validate the report

We will try to acknowledge valid reports promptly and coordinate a fix before public disclosure when possible.
